# План разработки MVP магазина приложений 

**Рабочее время:** 1 рабочий день = 8 часов. 1 неделя = 40 часов.

---

## Epic 1: Базовая архитектура и инфраструктура
**Описание:** Подготовка фундамента приложения, настройка зависимостей, навигации и базового UI-кита. Без этого этапа невозможно начать продуктовую разработку.
**Длительность:** 1 неделя (40 часов)

* **Task 1.1: Инициализация проекта и структура**
  * Описание: Первичная настройка пустого проекта. Добавление базовых библиотек и формирование пакетной структуры по слоям архитектуры.
  * Оценка: 6 часов
* **Task 1.2: Настройка Dependency Injection**
  * Описание: Интеграция DI-фреймворка. Настройка базовых модулей и графов зависимостей для связывания архитектурных слоев приложения.
  * Оценка: 6 часов
* **Task 1.3: Настройка Compose Navigation**
  * Описание: Интеграция навигации. Проектирование графа переходов между главными экранами приложения.
  * Оценка: 6 часов
* **Task 1.4: Разработка базового UI-кита**
  * Описание: Формирование дизайн-системы приложения. Настройка цветовой палитры, типографики и создание базовых переиспользуемых UI-компонентов.
  * Оценка: 8 часов
* **Task 1.5: Подготовка классов Clean Architecture**
  * Описание: Разработка фундаментальных классов, включая обертки для обработки результатов выполнения операций и базовые классы для управления состоянием UI.
  * Оценка: 6 часов
* **Task 1.6: Настройка статического анализа кода**
  * Описание: Подключение и базовая настройка инструментов статического анализа для автоматического контроля качества и стиля кода в проекте.
  * Оценка: 8 часов

## Epic 2: Каталог приложений (Список)
**Описание:** Реализация главного экрана со списком доступных приложений. Пользователь должен видеть список с базовой информацией.
**Длительность:** 1 неделя (40 часов)

* **Task 2.1: [Domain] Сущности и UseCase-ы списка**
  * Описание: Проектирование domain-слоя для каталога. Описание модели приложения и контрактов бизнес-логики для получения данных.
  * Оценка: 6 часов
* **Task 2.2: [Data] Репозиторий и mock-данные**
  * Описание: Реализация data-слоя. Создание репозитория и источника данных, эмулирующего задержку сети и возвращающего тестовый список приложений.
  * Оценка: 6 часов
* **Task 2.3: [Presentation] Настройка AppListViewModel**
  * Описание: Разработка ViewModel экрана списка. Реализация логики управления состояниями: загрузка, успешное получение данных и обработка ошибок.
  * Оценка: 6 часов
* **Task 2.4: [Presentation] UI карточки приложения**
  * Описание: Разработка UI-компонента отдельной карточки для списка, отображающего основную информацию.
  * Оценка: 8 часов
* **Task 2.5: [Presentation] UI экрана списка**
  * Описание: Сборка главного экрана. Настройка "ленивого" списка для вывода каталога с привязкой к реактивным состояниям из ViewModel.
  * Оценка: 8 часов
* **Task 2.6: [Presentation] Pull-to-refresh и обработка ошибок**
  * Описание: Интеграция механизма ручного обновления списка (свайп вниз) и добавление UI-элементов для информирования пользователя об ошибках.
  * Оценка: 6 часов

## Epic 3: Детальная карточка приложения
**Описание:** Реализация экрана с подробной информацией о конкретном приложении при клике на него в списке.
**Длительность:** 1 неделя (40 часов)

* **Task 3.1: [Domain] UseCase-ы детальной информации**
  * Описание: Расширение domain-модели для детального экрана. Добавление дополнительных полей и логики запроса подробной информации о приложении.
  * Оценка: 6 часов
* **Task 3.2: [Data] Расширение источника данных**
  * Описание: Адаптация репозитория и mock-источника для возврата полной информации о конкретном приложении по его id.
  * Оценка: 6 часов
* **Task 3.3: [Presentation] Настройка AppDetailsViewModel**
  * Описание: Разработка ViewModel детального экрана. Реализация логики загрузки подробной информации на основе переданного аргумента навигации.
  * Оценка: 6 часов
* **Task 3.4: [Presentation] UI экрана деталей**
  * Описание: Верстка экрана деталей. Размещение шапки приложения, блока с текстовым описанием и горизонтальной галереи скриншотов (как пример).
  * Оценка: 8 часов
* **Task 3.5: [Presentation] Интеграция переходов**
  * Описание: Связывание экранов. Настройка передачи id приложения при переходе из списка и корректная обработка возврата назад.
  * Оценка: 6 часов
* **Task 3.6: [QA] Покрытие Unit-тестами**
  * Описание: Написание базовых тестов для проверки корректности работы бизнес-логики и изменения состояний во ViewModel.
  * Оценка: 8 часов

## Epic 4: Установка и удаление приложений
**Описание:** Ключевой бизнес-функционал. Возможность инициировать установку, видеть прогресс и удалять приложение. (В рамках MVP — эмуляция процесса).
**Длительность:** 1 неделя (40 часов)

* **Task 4.1: [Domain] Модель состояний и UseCase-ов установки**
  * Описание: Проектирование бизнес-логики управления приложениями. Описание всех состояний жизненного цикла (установка/удаление) и соответствующих UseCase-ов.
  * Оценка: 6 часов
* **Task 4.2: [Data] Менеджер состояния установок**
  * Описание: Реализация локального кеш-менеджера состояний. Имитация асинхронных процессов скачивания и установки с обновлением статусов в реальном времени.
  * Оценка: 8 часов
* **Task 4.3: [Presentation] Реактивное обновление ViewModels**
  * Описание: Обеспечение мгновенной реакции UI на изменение состояния приложений на всех экранах.
  * Оценка: 8 часов
* **Task 4.4: [Presentation] UI динамической кнопки действия**
  * Описание: Разработка умного компонента кнопки, который меняет свой внешний вид и поведение в зависимости от текущего статуса приложения.
  * Оценка: 8 часов
* **Task 4.5: [Presentation] Интеграция бизнес-флоу**
  * Описание: Внедрение полного сценария установки/удаления на экранах списка и деталей. Связывание UI-событий с бизнес-логикой.
  * Оценка: 6 часов
* **Task 4.6: [QA] Ручное тестирование сценариев**
  * Описание: Проверка работоспособности ключевых пользовательских путей. Тестирование переходов состояний приложения от начала скачивания до полного удаления.
  * Оценка: 4 часа
